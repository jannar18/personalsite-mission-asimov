## Week 1: Simple Game (Tic Tac Toe)

### Day 1 — Simple Game
- Git (clone, branch, commit, push, PR, merge)
- GitHub (fork, pull requests)
- Homebrew (package manager)
- Bun (runtime + test runner)
- TypeScript (game logic)
- React (UI, state management)
- HTML tables (grid display)
- VSCode (Source Control panel)
- Unit testing (bun test)

### Day 4 — Real-time Game + Deployment
- WebSockets (express-ws, real-time client-server communication)
- Event subscriptions (per-game-ID channels)
- Render (cloud deployment)
- HMR (Hot Module Replacement — learned to configure/disable)

### Day 3 — Multi-Game Server
- UUID (unique game IDs)
- Test-Driven Development (TDD)
- REST API design (multiple endpoints: GET /games, POST /create, GET /game/:id, POST /move)
- In-memory data structures (multi-game state management)
- Game lobby UI pattern (list → select → play)

### Day 2 — Styling & Server/Client Architecture
- CSS (box model, flexbox, mobile-first)
- Vite-Express (combined dev server)
- Express (GET/POST endpoints, in-memory state)
- fetch() (client-server communication)
- Async/await + Promises
- Client-server architecture (game state on server)
